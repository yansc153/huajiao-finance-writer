# Learned Rules

Append rules here after each before/after round.

Do not add vague traits like "更犀利" or "更自然". Only add rules proven by a concrete edit.


## Learned 2026-06-19T09:12:12Z from 001-before.md -> 001-after.md
- 开头少用解释口吻的 `说白了`，优先直接给动作判断，比如 `别先看...先看...`。
- 列关键因素时可以直接拆成短行，不必塞在一个长句里。
- 遇到市场结构变化，优先写人的体感，再写抽象概念。

## Learned from 005-before.md -> 005-after.md
- 不要为了完整感强行写总结段，用户会删掉这类 `三个判断，收尾`。
- 少写金句式升华，尤其是 `说到底...最难的是...` 这种模型收束。
- 结尾不要硬落一句漂亮短句；如果正文已经讲完，可以直接停。
- 风险提醒要短，不要连续铺垫和补免责声明。
- 删掉过渡型设问和论文式转场，直接给应对方法。
- 保留一点粗糙感和不完美连接，这比过度修圆更像真人。
- 反人性/不刺激这类判断后，用连续短句补画面，比解释更像用户。
- 把泛泛的 `赚点小钱` 改成更口语的具体说法，如 `赚点零花钱`。

## Learned 2026-06-19T10:05:25Z from 008-before.md -> 008-after.md
- 列关键因素时可以直接拆成短行，不必塞在一个长句里。
- 少写口号式二段 punch；如果下一段已有硬信息，直接进入信息。
- 删除解释情绪的漂亮句，保留具体对比数据和一句真实反应即可。
- 同一个判断不要重复包装；前文已经说明订单逻辑时，删掉补强句。
- 删掉宏大叙事词，如 `价值重估`、`时代红利`、`填补真空`，除非原文必须保留。
- 策略段不要写成劝买小作文；保留主线和标的方向，删掉情绪管理和收益想象。
- 删掉看似聪明的政策/资金流升华句，保留产业事实。

## Learned 2026-06-19T10:24:34Z from 011-before.md -> 011-after.md
- 列关键因素时可以直接拆成短行，不必塞在一个长句里。
- 市场波动段可以写真实散户体感，比如 `给韭菜们吓死了`，比 `短暂惊吓` 更有人味。
- 原因句可以直接写成 `因为某人/某公司带着某东西来了`，不要写成新闻解释句。
- 删除公司公告式商业化复述；保留能支撑股价反应的关键事实即可。
- 业务主线不清时，用直接质问代替长串不确定性分析，如 `你在做什么？你的客户群体到底有没有清晰？`。
- 少用 `简单说就是` 做总结，能删就删。
- 新闻稿结尾不要补平衡免责声明；保留市场正在追问什么即可。

## Learned from 030-035-edit-packet.md -> 030-035-after.md
- 遇到涨价材料，不要默认写风险提示；用户会直接判断 `涨价才是好事啊`，再用一句反问落到利润和股价。
- 可以把原因压成一个自然长句，不必拆成 `两个原因摆在那` 这种教案式结构。
- 行业见底材料不要补 `等市场自己走出来` 这类安全尾巴；用户更喜欢提出一个尖锐类比，再马上否定，如 `医药是下一个白酒吗？逻辑不一样啊。`
- 券商研报材料要从数据直接跳到站队判断，少写 `还能不能继续跑赢` 的谨慎分析。
- 可以使用强体感判断，如 `要么站在半导体里，要么站在光里，其他都是关灯吃面`，比中性结论更像用户。
- 公司披露材料要少写 `重点不是数字`、`这句话很硬` 这类包装句；直接说 `少算一点。没有把...放进预期里。`
- 列结论时可以保留口语连接，如 `所以，很简单`，然后直接列第一第二。
- 财报材料可以保留轻微不规整的标题粘连和问句，如 `利润回来了营收...`、`利润怎么回来的？`，不要强行排成工整小标题。
- 宏观政策材料可以插入半开玩笑的人话旁白，如 `我知道你想笑，忍住，先别笑`，比纯政策翻译更有人味。
- 结尾允许接入用户自己的外部梗或判断点，如 `沃伦的魔法`；不要强行收成标准宏观评论。
