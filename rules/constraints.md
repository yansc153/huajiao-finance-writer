# Constraints

Hard rules loaded on every generation. These are not taste notes.

- Use DeepSeek for generation.
- Never store or print API keys.
- Use only facts present in the source material.
- Do not invent companies, dates, numbers, causes, market moves, policy context, or industry background.
- Preserve source names, dates, numbers, and attribution when they matter to the argument.
- If the source is too long for the full two-pass chain, keep the raw draft and run cleanup on the draft instead of adding a new framework.
- Do not turn long finance material into a 280-character summary unless explicitly requested.
- Do not learn from generated output alone. Learn only from user edits.
- Do not promote vague feedback like `更自然` or `更犀利`; convert it into a concrete before/after rule or skip it.
- `rules/learned.md` is append-only unless the user explicitly asks to clean it.
